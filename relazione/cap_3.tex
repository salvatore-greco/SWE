    Questa classe rappresenta la biblioteca. È caratterizzata da un nome ed un budget che ha a disposizione.\\

    \subsubsection{Book}
    Questa classe rappresenta un libro.\\
    Un libro può essere aggiunto o rimosso dal catalogo di tutti i libri della biblioteca da parte dell'amministratore della biblioteca. Se presente nel catalogo, può essere preso in prestito da un utente ed è identificato tramite ISBN, titolo e codice. Inoltre su questo è riportato anche il nome degli autori.\\
    Per la creazione di un oggetto Book viene usato il \textbf{BookBuilder}, mostrato nel Codice~\ref{lst:bookBuilder}, che implementa il design pattern del \textbf{Builder}.\\
    \begin{listing}[H]
        \begin{minted}[fontsize=\small]{java}
            public class Book {
            
                public static class BookBuilder {
                    private String ISBN;
                    private String title;
                    private ArrayList<String> authors;
                    private String code; //book code to identify multiple copies
            
                    public BookBuilder setISBN(String ISBN) {
                        this.ISBN = ISBN;
                        return this;
                    }
            
                    public BookBuilder setTitle(String title){
                        this.title = title;
                        return this;
                    }
            
                    public BookBuilder setAuthors(ArrayList<String> authors){
                        this.authors = authors;
                        return this;
                    }
            
                    public BookBuilder setCode(String code){
                        this.code = code;
                        return this;
                    }
            
                    public Book build(){
                        return new Book(this);
                    }
                }
            
                private Book(BookBuilder builder){
                    this.ISBN = builder.ISBN;
                    this.title = builder.title;
                    this.code = builder.code;
                    this.authors = builder.authors;
                }
            
                private String ISBN;
                private String title;
                private ArrayList<String> authors;
                private String code;
            
                public String getISBN() { return ISBN; }
                public String getTitle() { return title; }
                public ArrayList<String> getAuthors() { return authors; }
                public String getCode() { return code; }
            }
        \end{minted}

    Questa classe rappresenta un utente astratto, caratterizzato da nome, cognome ed email. La classe appunto è definita come astratta.     \subsubsection{Librarian}
    Questa classe rappresenta un bibliotecario, un utente con permessi aggiuntivi, che lavora all'interno della biblioteca. Può gestire gli eventi, creare/rinnovare tessere della biblioteca e monitorare i prestiti.\\
    
    \subsubsection{LibraryAdministrator}
    Questa classe rappresenta l'amministratore della biblioteca.\\

    \subsubsection{Card}
    Questa classe rappresenta una tessera della biblioteca associata a un utente.\\
    La tessera consente l'accesso ai servizi della biblioteca, come il prestito di libri e la prenotazione di risorse ed è identificata tramite id, data di emissione e data di scadenza.\\
    
    \subsubsection{Event}
    Questa classe rappresenta un evento organizzato dalla biblioteca.\\

    \subsubsection{Room}

    \subsubsection{EventRoom}

    \subsubsection{StudyRoom}

    L'\texttt{AuthService} è un'interfaccia che dichiara i principali metodi di autenticazione: \textbf{login, logout, register e resetPassword}, mostrati nel Codice~\ref{lst:authService}.\\
    \begin{listing}[H]

                public void logout();
                public void resetPassword(String email, String newPassword);

                public boolean isLogged();
            }
        \end{minted}
        \caption{Classe \texttt{AuthService}}
        \label{lst:authService}
    \end{listing}
    
    Questi vengono implementati in \texttt{ConcreteAuthService}. In particolare, il metodo \textbf{login} restituisce un tipo di utente diverso in base al ruolo dell'utente autenticato, come mostrato nel Codice~\ref{lst:login}.\\
    La gestione della sessione di un utente, è realizzato tramite il metodo \textbf{isLogged}, che verifica se un utente è autenticato o meno.\\

    \begin{listing}[H]
        \begin{minted}[fontsize=\small]{java}
        @Override
        public ControllerInterface login(String email, String password) throws UserNotFoundException {
            UserDTO userDTO = userDAO.getUserByEmail(email);
            ControllerFactory factory;
    
            if (!BCrypt.checkpw(password, userDTO.getPassword())) {
                throw new UserNotFoundException("Wrong username or password");
            }
    
            switch (userDTO.getRole()) {
                case role.libraryUser:
                    loggedUser = new LibraryUser.LibraryUserBuilder()
                            .setName(userDTO.getName())
                            .setSurname(userDTO.getSurname())
                            .setEmail(userDTO.getEmail())
                            .build();
                    factory = new LibraryUserControllerFactory();
                    break;
                case role.librarian:
                    loggedUser = new Librarian(userDTO.getName(), userDTO.getEmail(), userDTO.getSurname());
                    factory = new LibrarianControllerFactory();
                    break;
                case role.libraryAdministrator:
                    loggedUser = new LibraryAdministrator(userDTO.getName(), userDTO.getSurname(), userDTO.getEmail());
                    factory = new LibraryAdministratorControllerFactory();
                    break;
                default:
                    throw new RuntimeException("Role " + userDTO.getRole() + " does not exist ");
            }
            return factory.createController(loggedUser);
        }
        \end{minted}
        \caption{Metodo \textbf{login}}
        \label{lst:login}
    \end{listing}

    \subsubsection{ConnectionManager}
    La classe si occupa di gestire la connessione al database per tutte le classi DAO tramite il metodo \textbf{getConnection}.\\
    È implementata utilizzando il design pattern \textbf{Singleton}, con lo scopo di centralizzare la gestione delle connessioni.\\
    Per stabilire la connessione al database è necessaria la creazione di un file \texttt{.env}, con dentro inserite le credenziali per poterci accedere.\\
    Tutta la classe è mostrata in Codice~\ref{lst:connectionManager}.
    \begin{listing}[H]
        \begin{minted}[fontsize=\small]{java}
            public class ConnectionManager {
                private Connection connection;
                private static ConnectionManager instance = null;
            
                private ConnectionManager() throws SQLException, SQLTimeoutException {
                    this.connection = initializeConnection();
                }
            
                private Connection initializeConnection() throws SQLException {
                    Connection connection;
                    Properties props = getDBProprieties();
                    connection = DriverManager.getConnection(props.getProperty("url"), props);
                    return connection;
                }
            
                public static ConnectionManager getInstance() throws SQLException, SQLTimeoutException{
                    if (instance == null) instance = new ConnectionManager();
                    return instance;
                }
            
                public Connection getConnection(){
                    if (instance == null){
                        throw new IllegalStateException("getConnection must be called after getInstance");
                    }
                    try {
                        if (connection.isClosed()){
                            this.connection = initializeConnection();
                        }
                    } catch (SQLException e) {
                        e.printStackTrace();
                    }
                    return connection;
                }
            
                private Properties getDBProprieties(){
                    Dotenv dotenv = Dotenv.configure().directory("./src").load();
                    Properties props = new Properties();
                    props.setProperty("url", dotenv.get("DB_URL"));
                    props.setProperty("user", dotenv.get("DB_USER"));
                    props.setProperty("password", dotenv.get("DB_PASS"));
                    return props;
                }
                public String getDBUsername(){
                    return getDBProprieties().getProperty("user");
                }
            }
        \end{minted}
        \caption{Implementazione della classe \texttt{ConnectionManager}}
        \label{lst:connectionManager}
    \end{listing}

Di seguito, nel Codice~\ref{lst:esempio}, è riportato un esempio rappresentativo del funzionamento di un metodo appartenente al livello ORM. In particolare, viene mostrato un metodo della classe \texttt{RoomDAO}, che esegue una query SQL per recuperare informazioni dal database e costruire il corrispondente oggetto del dominio, in questo caso una \texttt{Room}.
    Il metodo utilizza JDBC per preparare ed eseguire la query, gestire il \texttt{ResultSet} e, in base ai dati ottenuti, istanziare dinamicamente l'oggetto adatto (\texttt{StudyRoom} o \texttt{EventRoom}), restituendolo al livello superiore dell'applicazione.
    \begin{listing}[H]
        \begin{minted}[fontsize=\small]{java}
            public Room getRoomByNumber(int number) {
                try {
                    Connection conn = ConnectionManager.getInstance().getConnection();
                    PreparedStatement stmt = conn.prepareStatement("SELECT * FROM room WHERE number = ?");
                    stmt.setInt(1, number);
                    ResultSet rs = stmt.executeQuery();
                    if(rs.next()) {
                        if (rs.getBoolean("is_study_room")) {
                            Room room = new StudyRoom(rs.getInt("number"), rs.getInt("seats"), 
                                getReservedSeatForRoomNumber(number));
                            stmt.close();
                            return room;
                        }
                        else {
                            ArrayList<Event> eventList = getEventForRoomNumber(number);
                            EventRoom eventRoom = new EventRoom(rs.getInt("number"), rs.getInt("seats"));
                            for (Event event : eventList) {
                                event.setPlace(eventRoom);
                            }
                            eventRoom.setScheduledEvents(eventList);
                            stmt.close();
                            return eventRoom;
                        }
                    }
                    stmt.close();
                } catch (SQLException e) {
                    e.printStackTrace();
                }
                return null;
            }
        \end{minted}
        \caption{Esempio di metodo DAO per il recupero di dati dal DB}
        \label{lst:esempio}
    \end{listing}

Esso descrive in modo concettuale il database. Ogni tabella è stata mappata a una classe nel package \texttt{DomainModel}, ad eccezione degli utenti, dove è stato scelto di accorparli in un entità sola fin dal principio dato che hanno molti attributi in comune, evitando di fare una generalizzazione per poi risolverla in fase di ristrutturazione; per differenziare il loro ruolo è stato scelto un flag \texttt{role}.
Per la relazione fra \texttt{User} e \texttt{Card} è stato scelto di farla 1:N con partecipazione opzionale da parte di \texttt{User} per dare modo di potersi registrare e richiedere la tessera in un secondo momento. \\
Sono state scelte fin da subito le chiavi primarie, evitando di farlo in fase di ristrutturazione; sono indicate col pallino pieno nell'attributo. 
Come già specificato in \ref{subsec:architettura} per il database è stato utilizzato \texttt{PostgreSQL}; sono stati usati i suoi data types, anche se non sono \textit{SQL} standard, come \texttt{duration}. Per la gestione dei ruoli e per avere una validazione anche lato database è stata definita un enumerazione:
\begin{code}
\mint{sql}|CREATE type role AS ENUM ('librarian', 'libraryUser', 'libraryAdministrator');|
\caption{enumerazione database}
\end{code}
\noindent Per inizializzare il database è stato creato il file \texttt{db.sql} contenente la definizione delle tabelle, che dovrà essere eseguito manualmente la prima volta; per avere dei dati iniziali da usare nei test è stato creato il file \texttt{data.sql} che invece è eseguito dalla classe \texttt{DBSeeder} nello schema \texttt{test} (\ref{subsec:struttura}).\\
